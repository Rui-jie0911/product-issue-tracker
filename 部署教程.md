# 产品问题追踪系统 — 部署教程（零基础版）

> 整篇教程大约需要 **30-40 分钟**完成。跟着步骤一步步来就行，不需要任何编程知识。

---

## 准备工作：注册三个免费账号

在开始之前，请先准备好以下三个账号（全部免费）：

| 平台 | 用途 | 注册地址 |
|------|------|----------|
| **GitHub** | 存放代码 | https://github.com 点击 Sign up |
| **Supabase** | 数据库 + 用户登录 + 照片存储 | https://supabase.com 点击 Start your project |
| **Vercel** | 网站托管（让App上线） | https://vercel.com 点击 Sign up，选择用 GitHub 账号登录 |

> **提示**：Vercel 直接用 GitHub 账号登录最方便，省一个账号。

---

## 第一步：把代码上传到 GitHub

### 1.1 下载并安装 GitHub Desktop

1. 打开 https://desktop.github.com/
2. 点击紫色的 **Download for Windows** 按钮
3. 下载完成后双击安装，一路点"下一步"即可
4. 安装完成后打开 GitHub Desktop，用你的 GitHub 账号登录

### 1.2 把项目代码导入 GitHub Desktop

1. 打开 GitHub Desktop
2. 点击菜单栏 **File → Add local repository**（添加本地仓库）
3. 点击 **Choose...** 按钮，浏览到文件夹：
   ```
   E:\历史粘贴\product-issue-tracker
   ```
4. 点击 **Add Repository**
5. 此时会提示 "This directory does not appear to be a Git repository"，点击 **create a repository** 链接
6. 在弹出的窗口中：
   - Name 填入：`product-issue-tracker`
   - 勾选 **Initialize this repository with a README** 不要勾
   - 点击 **Create Repository**

### 1.3 发布到 GitHub

1. 在 GitHub Desktop 中，点击右上角的 **Publish repository** 按钮
2. 在弹出的窗口中：
   - Name 保持 `product-issue-tracker`
   - **取消勾选** "Keep this code private"（设为公开，否则 Vercel 免费版无法部署）
   - 点击 **Publish Repository**
3. 等待上传完成（约1-2分钟）

> 上传完成后，打开 https://github.com/你的用户名/product-issue-tracker 就能看到你的代码了。

---

## 第二步：配置 Supabase（后端数据库）

### 2.1 创建 Supabase 项目

1. 打开 https://supabase.com 并登录
2. 点击右上角的绿色按钮 **New project**
3. 在创建项目页面填写：

   | 字段 | 填写内容 |
   |------|----------|
   | **Name** | `product-tracker`（随意取名） |
   | **Database Password** | 设置一个密码，**务必记下来**！（建议用浏览器记住密码功能） |
   | **Region** | 选择 **Northeast Asia (Tokyo)** 或 **Southeast Asia (Singapore)**，离中国最近 |

4. 点击绿色的 **Create new project** 按钮
5. 等待 2-3 分钟，直到页面显示 "Project is ready"

### 2.2 执行数据库建表脚本

1. 在 Supabase 项目页面，点击左侧菜单的 **SQL Editor**（SQL 编辑器）
2. 点击 **New query** 按钮
3. 现在回到你的电脑，用记事本打开文件：
   ```
   E:\历史粘贴\product-issue-tracker\supabase-setup.sql
   ```
4. **全选复制**（Ctrl+A → Ctrl+C）文件中的所有内容
5. 回到 Supabase 网页，**粘贴**（Ctrl+V）到 SQL 编辑器中
6. 点击右下角的绿色 **Run** 按钮
7. 看到 "Success" 提示即表示建表成功

### 2.3 创建照片存储桶

1. 在 Supabase 左侧菜单点击 **Storage**
2. 点击 **New bucket** 按钮
3. Bucket name 填入：`issue-photos`
4. **勾选** "Public bucket"（公开访问）
5. 点击 **Create bucket**

### 2.4 获取项目连接信息

1. 在 Supabase 左侧菜单点击 **Settings**（设置）→ **API**
2. 你会看到两个重要信息：
   - **Project URL**：类似 `https://xxxxxxxxxxxx.supabase.co`
   - **anon public key**：很长一串字母，以 `eyJ` 开头

3. 把这两个值记下来（或保持此页面打开，下一步要用）

## 第三步：配置项目环境变量

现在需要把 Supabase 的连接信息填入你的项目。

### 3.1 修改 .env 文件

1. 用**记事本**打开文件：
   ```
   E:\历史粘贴\product-issue-tracker\.env
   ```
2. 你会看到两行内容：
   ```
   VITE_SUPABASE_URL=https://your-project.supabase.co
   VITE_SUPABASE_ANON_KEY=your-anon-key-here
   ```
3. 替换为你的实际值：
   ```
   VITE_SUPABASE_URL=https://xxxxxxxxxxxx.supabase.co
   VITE_SUPABASE_ANON_KEY=eyJxxxxxxxxxxxxxxxxxxxxxx
   ```
4. 保存文件（Ctrl+S），关闭记事本

### 3.2 同步修改到 GitHub

1. 打开 GitHub Desktop
2. 你会在左侧看到 `.env` 文件有修改
3. 在左下角的 **Summary** 框中输入：`配置Supabase连接`
4. 点击 **Commit to main** 按钮
5. 点击右上角的 **Push origin**（推送到 GitHub）

---

## 第四步：部署到 Vercel（让 App 上线）

### 4.1 导入项目

1. 打开 https://vercel.com 并用 GitHub 账号登录
2. 进入仪表盘（Dashboard），点击 **New Project** 按钮
3. 在 "Import Git Repository" 界面，找到 `product-issue-tracker`，点击 **Import**
4. 如果找不到，点击 "Adjust GitHub App Permissions" 授权访问

### 4.2 配置部署参数

在配置页面中：

1. **Framework Preset** 会自动识别为 **Vite**（无需修改）
2. 展开 **Environment Variables** 部分，添加两个环境变量：

   | NAME | VALUE |
   |------|-------|
   | `VITE_SUPABASE_URL` | `https://xxxxxxxxxxxx.supabase.co`（你的 Supabase Project URL） |
   | `VITE_SUPABASE_ANON_KEY` | `eyJxxxxxxxxxxxxxxxxxxxxxx`（你的 Supabase anon key） |

   点击 **Add** 逐个添加。

3. 点击 **Deploy** 按钮

### 4.3 等待部署完成

1. 等待 1-2 分钟，Vercel 会自动构建和部署
2. 看到 🎉 **Congratulations!** 页面表示部署成功
3. 你会获得一个网址，类似：
   ```
   https://product-issue-tracker-xxxxx.vercel.app
   ```
4. 点击这个网址，你的 App 就上线了！

## 第五步：注册用户并开始使用

### 5.1 注册账号

1. 打开你的 Vercel 网址
2. 看到登录页面，点击 **立即注册**
3. 输入你的邮箱和密码（密码至少6位），点击 **注册**
4. 去邮箱查看验证邮件，点击确认链接
5. 回到登录页面，输入邮箱密码登录

### 5.2 添加第一个车型和批次

1. 登录后看到空白的首页
2. 点击右上角你的邮箱 → **数据维护**
3. 在"车型管理"标签页，点击 **添加车型**，输入车型名称（如"ZR380A旋挖钻机"）
4. 切换到"批次管理"标签页，选择刚创建的车型 → **添加批次**
5. 输入批次号（如"2026-001"）和台套数（如"5"）

### 5.3 记录第一个问题

1. 回到首页（点击左上角返回）
2. 展开车型 → 展开批次 → 点击 **新建问题**
3. 车型和批次自动选中，台套数和序号自动显示
4. 填写问题简述，可以点击**语音输入**按钮用说话代替打字
5. 点击 **记录问题** 完成

### 5.4 手机端使用

1. 在手机浏览器中打开同样的 Vercel 网址
2. 登录后看到同样的界面，已为手机触控优化
3. 点击新建问题时可以点击 **拍照** 按钮直接调起手机相机
4. 点击 **语音输入** 按钮说中文自动转文字
5. 在浏览器菜单中选择 **添加到主屏幕**，以后就像普通 App 一样打开

## 常见问题

**Q: 注册后收不到验证邮件？**
A: 检查垃圾邮件箱。如果还是没有，在 Supabase → Authentication → Settings 中检查邮件服务配置。

**Q: 照片上传失败？**
A: 检查 Supabase → Storage → issue-photos 桶的权限是 "Public"。在 SQL Editor 中执行建表脚本确保 RLS 策略正确。

**Q: 导出 Excel 后图片不显示？**
A: 确保照片已成功上传。导出时图片需要从网络加载，请保持网络连接。

**Q: Vercel 部署后网页打不开？**
A: Vercel 的域名在中国大陆可能访问较慢。如果长期无法访问，可以购买一个国内域名并配置 CDN。

**Q: 想让同事也能用？**
A: 把你的 Vercel 网址发给同事，让他们在浏览器中打开，注册账号后就可以了。所有人的数据都在同一个数据库中。

**Q: 数据安全吗？**
A: Supabase 使用 PostgreSQL 企业级数据库，数据有多重备份。定期导出 Excel 文件作为本地备份更保险。
